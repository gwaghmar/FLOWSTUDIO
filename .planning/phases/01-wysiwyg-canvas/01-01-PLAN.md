**Change 2: Mermaid canvas container — apply aspect-ratio CSS (WYSIWYG-01, WYSIWYG-03)**

Find the mermaid canvas `frameRef` div. It currently has this style object:
```typescript
style={{
  minWidth: `${frameW}px`,
  minHeight: `${frameH}px`,
  width: "max-content",
  height: "max-content",
  backgroundColor: bgColor,
  backgroundImage: showGrid ? ... : undefined,
  backgroundSize: showGrid ? "24px 24px" : undefined,
  fontFamily: selectedFont.cssValue,
}}
className="relative overflow-visible rounded-xl shadow-xl"
```

Replace with:
```typescript
style={{
  width: `${frameW}px`,
  aspectRatio: `${frameW} / ${frameH}`,
  backgroundColor: bgColor,
  backgroundImage: showGrid ? "radial-gradient(circle, #c8c8c8 1px, transparent 1px)" : undefined,
  backgroundSize: showGrid ? "24px 24px" : undefined,
  fontFamily: selectedFont.cssValue,
}}
className="relative overflow-hidden rounded-xl shadow-xl"
```

Key changes:
- `width: ${frameW}px` (fixed width = export width, subject to previewScale)
- `aspectRatio: ${frameW} / ${frameH}` (enforces the correct height from width)
- `overflow: hidden` → `overflow-hidden` class (was `overflow-visible`) — clips diagram content to the canvas boundary
- Remove `minWidth`, `minHeight`, `width: "max-content"`, `height: "max-content"` — these are replaced by the fixed width + aspect-ratio

**Change 3: Inner content div — center diagram in canvas**

Find the `ref={innerRef}` div immediately inside the mermaid frameRef:
```typescript
<div ref={innerRef} className="inline-flex min-h-full min-w-full items-start justify-start p-8" />
```

Change to:
```typescript
<div ref={innerRef} className="flex min-h-full w-full items-center justify-center p-8 overflow-hidden" />
```

This allows the SVG to be centered within the fixed-ratio canvas.

**Why these changes are safe for exports:**
- Mermaid PNG/SVG export uses `innerRef.current?.querySelector("svg")` directly — not dependent on frameRef dimensions — so this change does not affect 1:1 PNG/SVG export output.
- ZIP export temporarily overrides `node.style.width` and `node.style.height` anyway, so this also remains unaffected.
