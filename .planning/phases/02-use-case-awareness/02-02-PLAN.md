| Threat ID | Category | Component | Disposition | Mitigation Plan |

| T-02-03 | Tampering | `data.suggestedPresetId` applied to `setPresetId` | mitigate | Value originates from server-side whitelist validation in Plan 01 (VALID_PRESET_IDS). TypeScript type `SocialPresetId` on client is a second layer. No arbitrary string reaches `setPresetId`. |
| T-02-04 | Tampering | `handleUseCaseChange` — user-controlled UseCaseId | accept | Value comes from a static `<select>` with hardcoded options. No user text input. The four valid values are hardcoded in the JSX options. |
